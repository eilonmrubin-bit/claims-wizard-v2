## 5 פאזות

### פאזה 0 — מקורות

שלושה סוגי מקורות:

**קלט משתמש:**
- מטא-דאטא (שם תיק, הערות)
- פרטים אישיים (שנת לידה חובה)
- פרטי נתבע
- 3 צירי קלט: תקופות העסקה, דפוסי עבודה, מדרגות שכר
- יום מנוחה, מחוז, ענף
- תאריך הגשה
- קלט ותק
- טוגלים (הפעלה/השבתה של זכויות)
- ניכויי הפרשות (קלט)
- קלט ייחודי לזכויות

**מאגרי מידע סטטיים:**
- זמני שבת (CSV למחוז)
- תאריכי חגים (עברי → לועזי)
- שכר מינימום (היסטורי)
- ערך יום הבראה
- דמי נסיעות
- ניתן להרחבה (פורמט אחיד: effective_date + ערכים)

**הגדרות:**
- הגדרות OT (סיפים)
- הגדרות התיישנות (חלונות + הקפאות)
- בסיס היקף משרה (182)

### פאזה 1 — עיבוד ראשוני

סדר חובה. כל שלב תלוי בקודמו (מלבד ותק שמקבילי).

```
⓪ מתרגם הדפוסים
   קלט: work_patterns (3 רמות)
   פלט: work_patterns_resolved (הכל ברמה א)

① מארג
   קלט: employment_periods + work_patterns_resolved + salary_tiers + rest_day
   פלט: effective_periods, daily_records, employment_gaps, total_employment
   לוגיקה: ולידציה → sweep line → פירוט יומי → פערים

② OT stages 1–7
   קלט: daily_records, rest_day, district, shabbat_times, ot_config
   פלט: shifts (שעות + דרגות), weeks (week_type)
   לוגיקה: הרכבה → שיוך → סיווג שבוע → סף → OT יומי → OT שבועי → חלון מנוחה

⑤א ותק ענפי (מקבילי ל-②)
   קלט: daily_records, seniority_input, industry
   פלט: seniority_monthly, seniority_totals

③ אגרגטור
   קלט: shifts, effective_periods
   פלט: period_month_records (שעות), month_aggregates (שעות)
   לוגיקה: קיבוץ לפי תקופה×חודש, ואז לפי חודש

④ המרת שכר
   קלט: period_month_records (שעות), effective_periods (מדרגות), minimum_wage
   פלט: period_month_records (שדות שכר)
   לוגיקה: נרמול → מינימום → המרה דרך שעתי

⑤ב היקף משרה
   קלט: month_aggregates (שעות), job_scope_config
   פלט: month_aggregates (שדות job_scope)
   לוגיקה: regular_hours / base, מוגבל ל-1.0
```

### פאזה 2 — חישוב זכויות

כל הזכויות מקביליות. אין תלות בין זכות לזכות. כל זכות קוראת מ-SSOT וכותבת ל-rights_results.

**מוגדרות:**

```
שעות נוספות (stage 8 — תמחור)
   קלט: shifts, period_month_records.salary_hourly, right_toggles
   לוגיקה: tier × rest × salary_hourly → claim_amount (הפרש בלבד)

דמי חגים
   קלט: weeks, daily_records, effective_periods, salary_daily, holiday_dates, rest_day
   לוגיקה: סף 1/10 → week_type לכל חג → סינון → ערך יום
```

**טרם הוגדרו (דרוש סקיל לכל אחת):**

```
חופשה, פיצויי פיטורין, פנסיה, הבראה, דמי נסיעות, השלמת שכר
```

### פאזה 3 — פוסט-פרוססינג

סדר חובה. כל שלב תלוי בקודמו.

```

   קלט: rights_results, limitation_config, filing_date

   קלט: limitation_results, deductions_input

   פלט: claim_summary
   לוגיקה: אגרגציה סופית בלבד, אפס חישובים חדשים
```

## זרימת נתונים

### מאגרים סטטיים — מי צורך מה

```
שכר מינימום     → פאזה 1: המרת שכר (④)
זמני שבת        → פאזה 1: OT stages 1–7 (②)
תאריכי חגים     → פאזה 2: דמי חגים
ערך יום הבראה   → פאזה 2: הבראה (עתידי)
דמי נסיעות      → פאזה 2: נסיעות (עתידי)
```

כל מאגר סטטי הוא טבלת lookup עם effective_date. הזכות/המודול שצריך אותו שולף לפי תאריך — לא דרך המארג.

### SSOT — שכבות נתונים

```
שכבה 0: קלט גולמי (פאזה 0)
שכבה 1: תקופות אפקטיביות + פערים (מארג)
שכבה 2: אינדקס יומי + משמרות + שבועות (מארג + OT)
שכבה 3: סיכומי תקופה×חודש (אגרגטור + המרת שכר)
שכבה 4: סיכומי חודש + ותק (אגרגטור + היקף משרה + ותק)
שכבה 5: תוצאות זכויות (פאזה 2)
שכבה 6: תוצאות פוסט-פרוססינג (פאזה 3)
שכבה 7: סיכום תביעה (פאזה 3)
```

### עיקרון המתזמר

```
כל מודול = פונקציה טהורה (קלט → פלט)
רק המתזמר (pipeline.py) קורא מ-SSOT וכותב אליו
אין תלות ישירה בין מודולים — רק דרך SSOT
```
